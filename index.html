    Logic Explanation

    This application uses a persistent array called lootArray to store loot objects. Each object contains a name and value property. The addLoot() function validates user input using conditional if statements before pushing a new object into the array. A traditional for loop is used inside renderLoot() to dynamically generate list elements and calculate the running total. Another loop in splitLoot() calculates the total again before dividing it evenly among party members. Conditional logic prevents invalid inputs and handles edge cases such as empty loot or invalid party size.
